Maven은 Java 프로젝트의 빌드, 의존성 관리, 배포를 표준화해주는 빌드 도구이다.

## **Maven이란?**
{: .mt-5 .mb-2}
Maven은 프로젝트의 구조와 빌드 과정을 규칙 기반으로 관리한다.
`pom.xml` 파일에 필요한 설정을 선언하면, Maven이 필요한 작업을 자동으로 수행한다.

## **핵심 개념**
{: .mt-5 .mb-2}

#### **POM (Project Object Model)**
{: .mt-4 .mb-2}
프로젝트 정보와 의존성, 플러그인, 빌드 설정을 담는 핵심 파일이다.

#### **의존성(Dependency) 관리**
{: .mt-4 .mb-2}
라이브러리 버전과 전이 의존성을 자동으로 해결해준다.
필요한 라이브러리는 `pom.xml`에 선언만 하면 Maven Repository에서 받아온다.

#### **Build Lifecycle**
{: .mt-4 .mb-2}
Maven은 정해진 생명주기 단계로 빌드를 수행한다.
대표 단계: `validate -> compile -> test -> package -> verify -> install -> deploy`

#### **Repository**
{: .mt-4 .mb-2}
- Local Repository: 내 PC에 캐시되는 저장소
- Remote Repository: Maven Central, Nexus 같은 원격 저장소

## **자주 쓰는 명령어**
{: .mt-5 .mb-2}
- `mvn clean`: 이전 빌드 결과 삭제
- `mvn compile`: 소스 컴파일
- `mvn test`: 테스트 실행
- `mvn package`: JAR/WAR 패키징
- `mvn install`: 로컬 저장소에 아티팩트 설치

## **한 줄 정리**
{: .mt-5 .mb-2}
Maven은 "프로젝트 빌드와 의존성 관리를 선언형으로 표준화해주는 도구"다.
